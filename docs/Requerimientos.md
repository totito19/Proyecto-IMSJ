# Documento de Requisitos
## Sistema de Gestión de Educación Vial - IMSJ
---

**Descripción General:**
Sistema web para centralizar información de la sección Tránsito de la Intendencia Municipal de San José y facilitar la gestión de trámites vinculados a la libreta de conducir.

---
## Requerimientos Funcionales

| Código | Requerimiento Funcional |
|--------|------------------------|
| RF1 | El sistema debe permitir iniciar sesión con email y contraseña. |
| RF2 | El sistema debe permitir que los ciudadanos consulten anuncios y noticias de la sección Tránsito. |
| RF3 | El sistema debe permitir que los ciudadanos se agenden para la prueba de manejo. |
| RF4 | El sistema debe permitir que los ciudadanos agenden renovación de libreta de conducir. | 
| RF6 | El sistema debe permitir que los ciudadanos accedan a materiales de estudio para aspirantes. |
| RF7 | El sistema debe permitir que los ciudadanos consulten preguntas frecuentes. |
| RF8 | El sistema debe permitir publicar y administrar noticias visibles para el público por parte del personal de IMSJ. |
| RF9 | El sistema debe permitir definir período de vigencia de cada noticia. |
| RF10 | El sistema debe permitir cargar imagen de portada, galería de imágenes, texto y enlaces útiles en las noticias. |
| RF11 | El sistema debe permitir gestionar el estado de una noticia ej: como publicada o no-publicada. |
| RF12 | El sistema debe permitir cargar franjas disponibles para trámites normales. |
| RF13 | El sistema debe permitir cargar franjas disponibles para trámites urgentes. |
| RF14 | El sistema debe permitir cargar franjas disponibles para prueba de manejo. |
| RF15 | El sistema debe permitir visualizar la agenda por día. |
| RF16 | El sistema debe permitir visualizar la agenda por semana. |
| RF17 | El sistema debe permitir visualizar la agenda por mes. |
| RF18 | El sistema debe permitir administrar materiales de estudio. |
| RF19 | El sistema debe permitir mantener la sección de preguntas frecuentes. |

---
## Requerimientos No Funcionales

| Código | Requerimiento No Funcional |
|--------|---------------------------|
| RNF1 | Control de roles y permisos |
| RNF2 | Validación de entradas |
| RNF3 | Historial completo de acciones administrativas |
| RNF4 | Protección de datos personales |
| RNF5 | Usabilidad móvil (mobile first) |
| RNF6 | Accesibilidad básica para público general |
| RNF7 | Manejo de vigencia automática de noticias |
| RNF8 | Prevención de doble reserva de agenda |
| RNF9 | Confirmación visual de agenda para el ciudadano |
| RNF10 | Separación clara entre contenidos publicados y contenidos no-publicados |
